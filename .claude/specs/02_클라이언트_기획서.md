# 클라이언트 기획서 — 네이티브 & 웹 플레이어 (MVP / v1)

> 데이터 모델·용어·API 원칙은 **공통 토대 문서(`00`)** 를 따른다. 충돌 시 그쪽이 우선.

---

## 1. 목적 & 범위

서버의 **완전 독자 API(v1)** 에 붙는 재생 클라이언트.

**플랫폼**
- iOS, macOS 네이티브 (공유 코드베이스 지향)
- 웹

**프로토타입 1차 범위**: iOS / macOS 네이티브 + 웹.

라이선스는 서버와 별개다. 클라이언트는 네트워크 API로 통신하는 독립 프로그램이라 서버의 AGPL이 전염되지 않는다.

---

## 1-A. 기술 스택 (확정)

**네이티브 (iOS / macOS) — 공유 코드베이스**
- **Swift + SwiftUI 멀티플랫폼** (한 코드베이스로 iOS·macOS)
- 오디오 엔진 / EQ / 크로스페이드: **AVAudioEngine** (`AVAudioUnitEQ`, 믹서 노드)
- 비디오: **AVPlayer**
- 잠금화면 / 미디어 컨트롤: **MPNowPlayingInfoCenter + MPRemoteCommandCenter**
- 다이나믹 아일랜드 / Live Activity: **ActivityKit** (영상 재생 불가 → 정지 이미지만)
- 백그라운드 재생: **AVAudioSession**
- 서버 클라이언트는 서버의 **OpenAPI에서 생성**

**웹**
- **React + TypeScript + Vite** (서버가 TS라 타입·API 클라이언트 공유 용이; Svelte도 대안)
- 오디오/EQ/크로스페이드: **Web Audio API** (BiquadFilter, gain 노드)
- OS 미디어 컨트롤: **Media Session API**

> 네이티브에서 Flutter/React Native를 쓰지 않는 이유: 핵심 기능(EQ, 다이나믹 아일랜드, 잠금화면, 백그라운드 오디오)이 전부 네이티브 API라 결국 브릿지로 다시 깔아야 하므로 크로스플랫폼 이점이 사라진다.

---

## 2. 책임 경계 — 클라이언트가 가지는 것

클라가 소유: 재생 엔진, 큐/셔플/반복, **크로스페이드·이퀄라이저 같은 클라측 DSP**(디코딩된 오디오가 로컬에 있어야 함),
오디오↔비디오 전환 UX, UI 전체, OS 네이티브 통합.

라이브러리 상태의 진실은 **서버**에 있다. 클라는 그것을 캐시하고 표시할 뿐, 권위 있는 상태를 자체 생성하지 않는다.

---

## 3. 서버 연동

- **인증**: 서버 주소 입력 + 로그인 → 토큰 저장 및 자동 갱신.
- **라이브러리**: 델타 동기화(`since` 커서)로 로컬 캐시를 유지하고, 실시간 이벤트로 갱신한다.
- **스트리밍**: `track_id` + `media_kind` + (네트워크별) target bitrate로 요청. Range로 seek.
- **단일 서버 연결 기준**(v1). 멀티 서버/계정 전환은 로드맵.

---

## 4. 플레이어 (핵심 차별점)

### 4.1 통합 플레이어
소스 출처(로컬/외부)와 무관하게 동일한 재생 UX.

### 4.2 오디오 ↔ 비디오 전환
- video source가 있는 트랙에만 전환 버튼을 노출한다.
- 전환 시 재생 위치 유지는 **공통 토대 §5의 오프셋 규칙**으로 처리한다 (단순 위치 복사 아님).
- 백그라운드 전환 시 비디오 → 오디오로 자동 전환한다 (iOS는 일반 비디오의 백그라운드 재생 불가).

### 4.3 미니 / 풀스크린
- **미니 플레이어**: 하단 고정. 아트워크·제목·아티스트·재생 컨트롤, 전환 버튼, 위로 스와이프 시 풀스크린.
- **풀스크린(오디오)**: 큰 아트워크 + 컨트롤 + 가사 탭.
- **풀스크린(비디오)**: 풀스크린 영상 + 오버레이 컨트롤.

### 4.4 큐 / 셔플 / 반복
- 큐 편집(드래그 순서 변경, 추가/제거).
- 셔플(전체/앨범/플레이리스트 단위), 반복(전체/한곡/끄기).
- 큐·재생기록 상태를 서버와 동기화.

### 4.5 크로스페이드 (클라 DSP)
- 0~12초 설정. 두 플레이어 인스턴스로 페이드 인/아웃.

### 4.6 이퀄라이저 (클라 DSP)
- 프리셋 + 커스텀. iOS/macOS는 AVAudioEngine 계열, 웹은 Web Audio API.

### 4.7 ReplayGain 적용
- 서버가 제공한 gain 값으로 볼륨을 정규화한다 (트랙/앨범 모드 선택).

## 5. 라이브러리 & 탐색 UI

- 아티스트 / 앨범 / 트랙 브라우징.
- 앨범의 여러 버전을 한 페이지에서 묶어 표시.
- **커스텀 아티스트도 Last.fm 아티스트와 동일한 UI에서 혼용** (차별점).
- 검색은 서버 검색 API를 사용.

---

## 5-A. 재생 통계 (사용자)

- 최다 재생 트랙/아티스트/앨범, 기간 필터(주간/월간/전체), 총 청취 시간.
- 최근 추가 / 최근 재생.
- 서버 통계 API(`/stats/me`)를 사용.

---

## 6. 가사

- 풀스크린 플레이어의 가사 탭.
- LRC 싱크 가사면 현재 재생 위치에 맞춰 하이라이트.
- 소스: 로컬 LRC + 유저 입력(서버가 제공). 외부 가사 연동은 로드맵.

## 7. 네이티브 통합

### iOS
- **잠금화면**: 아트워크·제목·아티스트, 재생/일시정지/이전/다음, seek 바 (Now Playing Info + Remote Command).
- **다이나믹 아일랜드 / Live Activity**: 미니 아트워크 + 재생 진행바.
  **영상 재생은 불가하므로 비디오 재생 중엔 정지 썸네일/이미지만 표시한다.** 탭하면 풀스크린 플레이어로 이동.
- **백그라운드 오디오 재생**: 앱 백그라운드 전환·타 앱 사용 중에도 재생·컨트롤 유지.

### macOS
- Now Playing(Control Center) / 미디어 키 연동.
- 메뉴바 미니 플레이어.
- 백그라운드 재생.

## 8. 웹 클라이언트 (서버가 정적 서빙)

서버가 이 웹 앱을 번들로 자기 주소에서 서빙한다 → 컨테이너 하나로 플레이어 + 관리가 끝난다(Navidrome 방식). 별도 웹 앱을 두 개 만들지 않고, **하나의 앱 안에서 role로 화면을 게이팅**한다.

### 8.1 일반 사용자 (플레이어)
- 브라우징 + 검색 + 재생(통합 플레이어, 오디오↔비디오 포함)을 갖춘 **풀 플레이어**.
- OS 잠금화면/다이나믹 아일랜드 같은 네이티브 통합은 해당 없음. 대신 Media Session API로 OS 미디어 컨트롤 일부 연동.

### 8.2 관리자 대시보드 (role = admin 게이팅)
서버에 이미 있는 관리 API의 앞단 UI다. raw API로는 사실상 불가능한 작업(매핑 확인, 병합 등)을 위해 필수.
- **노래 업로드**: 다중 파일 드래그&드롭 + 진행상태. 업로드 직후 인라인 메타데이터 보정(제목/아티스트/앨범) 후 커밋.
- **이름 변경**: 트랙/앨범/아티스트 이름 변경(override 적용).
- **커스텀 아티스트 등록**: 외부 조회 없이 **수동 입력 폼**으로 생성(버튜버·커버·인디 등). 하드코딩성 수동 등록으로 충분.
- **아트워크 업로드/교체**.
- 스캔 트리거·진행상태, **매핑 확인 대기 큐**(승인/거부/수동연결), **중복 병합**, 유저 초대·관리, 서버 설정.

### 8.3 모니터링 & 통계 (관리자)
- **활성 스트림**: 현재 누가 무엇을 재생 중인지 실시간.
- **트래픽**: 대역폭(유저별/전체), 요청량, 트랜스코딩 부하·캐시 적중률, 기간 필터.
- **스토리지**: 라이브러리/캐시/업로드 사용량 분해.
- 서버 헬스 상태.

### 8.4 운영 유틸리티 (관리자)
- **라이브러리 무결성 점검**(사라진 파일·고아 메타·깨진 아트워크) 실행·결과 표시.
- **트랜스코딩 캐시 비우기**.
- **백업/복원**(메타데이터·설정).
- **로그 뷰어**.

### 8.5 미리듣기(임시) 플레이어
- 대시보드 어디서든 **트랙을 즉시 재생해 확인**하는 인라인 미니 플레이어. 풀 플레이어로 진입하지 않고 그 자리에서 재생.
- 주 용도: 업로드 직후 재생 검증, 매핑 확인 큐에서 "이 소스가 맞는지" 확인, 중복 후보 청취 비교.

> 공유 링크 미리듣기 전용(비로그인) 모드는 공유 기능과 함께 로드맵.
