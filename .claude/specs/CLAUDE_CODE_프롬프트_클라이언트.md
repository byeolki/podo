# Claude Code 프롬프트 — 클라이언트 앱

아래 내용을 그대로 Claude Code에 전달한다.
(저장소에 `00_공통토대.md`, `02_클라이언트_기획서.md`를 함께 두고 시작할 것. 서버 OpenAPI 스펙이 있으면 같이 제공.)

> 클라이언트는 **네이티브(iOS/macOS)**와 **웹** 두 타깃이다. 별도 저장소로 나눠도 되고, 모노레포로 묶어도 된다.
> 권장 순서: **웹을 먼저** 만들어 서버 API를 빠르게 검증한 뒤 네이티브로 넘어간다(네이티브는 OS 통합 비용이 큼).

---

## 역할

self-hosted 음악 스트리밍 서비스의 **재생 클라이언트**를 처음부터 구현한다.
같은 디렉토리의 `00_공통토대.md`(데이터 모델·API 계약)와 `02_클라이언트_기획서.md`(클라 범위)를 **정본 명세로 삼는다.**
명세와 충돌하면 명세를 따르고, 모호하면 질문한다.

## 핵심 원칙 (반드시 준수)

- 라이브러리 상태의 **진실은 서버에 있다.** 클라는 델타 동기화로 캐시하고 표시할 뿐, 권위 있는 상태를 자체 생성하지 않는다.
- 한 곡(Track)이 여러 소스를 가지며, **오디오↔비디오 전환 시 위치 유지는 `00_공통토대.md` §5 오프셋 규칙**으로 처리한다(단순 currentTime 복사 금지).
- **크로스페이드·이퀄라이저는 클라 측 DSP**다(디코딩된 오디오 필요).
- 서버 API 클라이언트는 가능하면 **OpenAPI에서 생성**한다.

---

## 타깃 A — 네이티브 (iOS / macOS)

**스택**: Swift + SwiftUI 멀티플랫폼(한 코드베이스로 iOS·macOS).
**MVP 범위**:
- 서버 연결/로그인(토큰 저장·갱신), 델타 동기화 + 실시간 이벤트 수신
- 라이브러리 브라우징(아티스트/앨범/트랙, 앨범 버전 묶음, 커스텀 아티스트 혼용) + 검색
- 통합 플레이어: 미니/풀스크린, 큐 편집, 셔플/반복
- **오디오↔비디오 전환**(video 소스 있는 트랙만 버튼; 백그라운드 시 비디오→오디오 자동) — AVPlayer
- 크로스페이드(0~12초)·이퀄라이저(프리셋+커스텀) — **AVAudioEngine**(`AVAudioUnitEQ`)
- ReplayGain 적용
- 가사 표시(LRC 싱크 하이라이트)
- 네이티브 통합:
  - iOS 잠금화면(아트워크/메타/컨트롤/seek) — MPNowPlayingInfoCenter + MPRemoteCommandCenter
  - **다이나믹 아일랜드/Live Activity(ActivityKit)** — **영상 재생 불가, 정지 이미지만**
  - 백그라운드 오디오(AVAudioSession)
  - macOS: Now Playing/미디어 키, 메뉴바 미니 플레이어

## 타깃 B — 웹 (서버가 정적 서빙)

**스택**: React + TypeScript + Vite (Svelte도 가능). **정적 빌드 산출물을 서버가 자기 주소에서 서빙**하므로 그 전제로 빌드한다(SPA, API는 같은 오리진).
**MVP 범위**: 하나의 앱 안에서 **role로 화면을 게이팅**한다(플레이어 + 관리자 대시보드 통합).

일반 사용자(플레이어):
- 로그인 + 브라우징 + 검색 + 통합 플레이어(오디오↔비디오 포함)
- EQ/크로스페이드는 **Web Audio API**(BiquadFilter, gain 노드)
- OS 미디어 컨트롤은 **Media Session API**
- 네이티브 통합(잠금화면/다이나믹 아일랜드)은 해당 없음

관리자 대시보드(role = admin):
- **노래 업로드**(다중 드래그&드롭 + 진행상태, 업로드 직후 인라인 메타데이터 보정 후 커밋)
- **이름 변경**(트랙/앨범/아티스트, override), **커스텀 아티스트 수동 등록 폼**(외부 조회 없이), **아트워크 업로드/교체**
- 스캔 트리거·상태, **매핑 확인 대기 큐**(승인/거부/수동연결), **중복 병합**, 유저 관리, 서버 설정
- **미리듣기(임시) 인라인 플레이어**: 대시보드 어디서든 트랙을 즉시 재생해 검증(업로드 직후·매핑 큐·중복 비교용). 풀 플레이어 진입 없이 그 자리에서 재생.
- **모니터링 & 통계**: 활성 스트림(실시간), 트래픽(대역폭·요청·트랜스코딩 부하), 스토리지 분해, 서버 헬스.
- **운영 유틸리티**: 라이브러리 무결성 점검, 트랜스코딩 캐시 비우기, 백업/복원, 로그 뷰어.

일반 사용자 통계:
- **재생 통계**: 최다 재생 트랙/아티스트/앨범, 기간 필터, 총 청취 시간, 최근 추가·재생 (`/stats/me`).

**공통 비범위(만들지 말 것)**: 오프라인 캐싱, 스마트 플레이리스트 UI, 관계도 시각화, 통계 대시보드, 공유 링크 미리듣기, 외부 가사, Android, 멀티 서버 전환.

## 작업 규칙 (공통)

- **Docker 우선.** 웹 클라이언트는 `Dockerfile` + compose로 빌드·서빙을 처음부터 구성한다(유지보수·배포 일관성). 네이티브는 Docker 대상이 아니므로 빌드/실행 방법을 README에 정리한다.
- **오픈소스 참고.** 플레이어 UX(미니/풀스크린 전환, 큐, 잠금화면 동작 등)는 Plexamp, Amperfy, Jellyfin 클라이언트 같은 성숙한 앱을 **참고**한다. 단 **코드를 복사하지 말 것**(라이선스 문제) — 동작/UX 패턴만 참고하고 구현은 직접 한다.
- **주기적 커밋.** 의미 있는 단위마다 자주 커밋한다. 거대한 단일 커밋을 만들지 않는다.
- **커밋 컨벤션**: Conventional Commits (`feat:`, `fix:`, `refactor:`, `chore:`, `docs:`, `test:` …)를 지킨다.
- **언어**: 커밋 메시지와 README는 **영어로** 작성한다.
- **코드 주석은 작성하지 않는다.** 코드는 이름과 구조로 의도를 드러낸다.
- **커밋에 본인 크레딧을 넣지 않는다.** `Co-authored-by`, "Generated with Claude Code" 등 AI 작성 표시를 커밋 메시지·푸터·README 어디에도 넣지 말 것.
- README에는 설정·실행 방법(웹은 docker compose, 네이티브는 빌드 절차)과 서버 연결 방법을 문서화한다.
